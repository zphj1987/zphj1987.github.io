openeuler不识别cmake_build

背景

在openeuler上面打包rpm的时候，有的src.rpm内部调用了%cmake_build的宏变量，openeuler的默认cmake3.22版本不支持，可以通过修改宏变量来实现可用

修改

修改文件路径为: /usr/lib/rpm/macros.d/macros.cmake

通过上面的修改后，即可适配高版本的编译
